KDA 심의 상태 모니터링

대한치과의사협회 의료광고 심의 상태 자동 모니터링 시스템

1. 개요

이 프로그램은 대한치과의사협회(KDA) 의료광고심의위원회의 심의 상태를 자동으로 모니터링하고, 상태 변경 시 Slack으로 알림을 발송하는 Windows 데스크톱 애플리케이션입니다.

주요 기능

  • KDA 심의 목록 자동 수집 및 상태 추적
  • 심의 상태 변경 감지 및 Slack 알림
  • 스케줄 기반 자동 수집 (매일 특정 시간 또는 일정 간격)
  • 즐겨찾기 및 검색/필터링 기능
  • 자동 업데이트

시스템 요구사항

항목 요구사항
운영체제 Windows 10/11 (64bit)
네트워크 인터넷 연결 필요 (dentalad.or.kr 접속)
필수 정보 KDA 로그인 계정 (빌드 시 아이디 주입, 비밀번호는 설정에서 입력)

2. 설치

관리자로부터 전달받은 설치 파일을 실행합니다.

  1. 설치 파일(group-d-kda-review-monitor-setup-x.x.x.exe) 실행
  2. 설치 완료 후 자동으로 프로그램 실행
참고 프로그램은 사용자 폴더에 설치되며, 관리자 권한이 필요하지 않습니다.

3. PIN 입력

프로그램 실행 시 4자리 PIN을 입력해야 합니다.

주의 PIN은 관리자에게 문의하세요. PIN은 GitHub 저장소에서 중앙 관리되며, 관리자가 변경하면 즉시 모든 버전에 적용됩니다.

4. 초기 설정

처음 사용 시 KDA 로그인 비밀번호를 입력해야 합니다.

입력 항목

항목 설명
아이디 빌드 시 주입됨 (변경 불가, 읽기 전용)
비밀번호 KDA 로그인 비밀번호 (Windows 자격증명에 안전하게 저장)
보안 비밀번호는 Windows 자격증명 관리자에 암호화되어 저장됩니다. 프로그램 외부에서 접근할 수 없습니다.

로그인 검증

설정 완료 버튼을 클릭하면 자동으로 KDA 사이트 로그인을 검증합니다. 로그인 성공 시 대시보드로 이동합니다.

5. 대시보드

로그인 후 메인 화면에서 심의 목록과 상태를 확인할 수 있습니다.

화면 구성

  • 상단 헤더: 수집 정보 (마지막 수집 시간, 이전 수집 시간)
  • 필터: 즐겨찾기, 상태 변경만 보기
  • 검색창: 병원명, 담당자명, 전화번호로 검색
  • 심의 테이블: 병원명, 담당자, 전화번호, 매체, 현재/이전 상태, 승인번호

상태 변경 표시

이전 수집 데이터와 비교하여 상태가 변경된 항목은 파란색 배경으로 강조 표시됩니다.

컬럼 설명
병원명 광고주 병원 이름
담당자 담당자 이름
전화번호 연락처
매체 광고 매체 (네이버, 카카오 등)
현재 상태 가장 최근 수집된 심의 상태 (심의 중, 승인, 반려 등)
이전 상태 직전 스냅샷의 심의 상태
승인번호 심의 승인 시 부여되는 번호

즐겨찾기

자주 확인하는 심의 항목은 별 아이콘을 클릭하여 즐겨찾기에 추가할 수 있습니다.

6. 데이터 수집

KDA 사이트에서 심의 목록을 자동으로 가져와 현재 상태를 저장하고, 이전 데이터와 비교하여 변경사항을 감지합니다.

수집 방식: 스케줄 자동 수집

이 프로그램은 설정에서 지정한 스케줄에 따라 완전 자동으로 데이터를 수집합니다. 사용자가 별도로 버튼을 누르거나 조작할 필요가 없습니다.

수집 모드 설정 방법 동작 방식 권장 용도
매일 특정 시간 시간 입력 (HH:mm 형식) 매일 지정한 시간에 1회 자동 수집
예: 매일 오전 9시, 매일 오후 6시		 업무 시작 전 또는 종료 후 1일 1회 확인
일정 간격 간격 선택
(30분~6시간)		 지정한 시간 간격으로 반복 수집
프로그램 실행 후 첫 수집 → 타이머 시작		 실시간 모니터링 (2~3시간 간격 권장)
스케줄 작동 조건 스케줄은 프로그램이 실행 중일 때만 작동합니다. 컴퓨터가 절전 모드이거나 프로그램을 종료하면 수집되지 않습니다.

선택 가능한 수집 간격

간격 1일 수집 횟수 적합한 상황
30분 약 48회 긴급 심의 진행 중 (단기간 집중 모니터링)
1시간 24회 실시간 상태 추적 필요 시
2시간 12회 권장 - 업무 시간 내 충분한 모니터링
3시간 8회 주요 시간대만 체크
4시간 6회 여유 있는 모니터링
5시간 약 5회 하루 수 차례 확인
6시간 4회 최소 모니터링 (오전/오후 2회씩)

자동 수집 과정

스케줄 시간이 되면 다음 작업이 자동으로 순차 실행됩니다:

단계 작업 소요 시간
1 저장된 자격증명으로 KDA 사이트 자동 로그인 약 2-3초
2 심의 목록 페이지 접속 및 데이터 추출 약 3-5초
3 데이터베이스에 현재 데이터 저장 1초 미만
4 이전 데이터(스냅샷)와 비교하여 상태 변경 감지 1초 미만
5 변경된 항목이 있으면 Slack 알림 발송 1초 미만
6 현재 데이터를 스냅샷으로 복사 (다음 비교 기준) 1초 미만
자동화 완료 모든 과정이 백그라운드에서 자동으로 실행되므로, 프로그램을 실행만 해두면 지정한 시간에 알아서 수집하고 알림을 보냅니다.

처음 수집 시 동작

처음 수집 시에는 이전 데이터가 없으므로 다음과 같이 동작합니다:

  1. KDA에서 데이터 수집
  2. 현재 데이터로 저장
  3. 스냅샷 자동 생성 (비교 기준 마련)
  4. Slack 알림 없음 (비교 대상이 없으므로)

두 번째 수집부터는 첫 수집 시 생성된 스냅샷과 비교하여 변경사항을 감지하고 알림을 발송합니다.

중요 프로그램을 처음 설치한 후 반드시 스케줄을 설정하세요. 스케줄 없이는 데이터 수집이 전혀 이루어지지 않습니다.

7. 이전 데이터 기준 (스냅샷)

상태 변경을 감지하기 위해 이전 수집 데이터를 스냅샷으로 저장합니다.

스냅샷이란?

스냅샷은 특정 시점의 심의 상태를 저장한 것으로, 현재 데이터와 비교하여 변경 사항을 감지하는 기준이 됩니다.

스냅샷 갱신 정책

이 프로그램은 "스케줄 수집 시 자동 갱신" 정책을 사용합니다.

수집 시점 스냅샷 갱신 설명
처음 수집 자동 생성 스냅샷이 비어있으면 무조건 생성 (비교 기준 마련)
스케줄 자동 수집 자동 갱신 현재 데이터를 스냅샷으로 복사 (새로운 비교 기준)
매번 수집할 때마다 스냅샷 업데이트

동작 시나리오

시나리오 1: 처음 사용

순서 시간 동작 스냅샷 상태
1 첫 수집 데이터 수집 → 스냅샷 자동 생성 ✅ 생성됨 (비교 기준 마련)
2 두 번째 수집 데이터 수집 → 스냅샷과 비교 → 변경 감지 → 스냅샷 갱신 ✅ 업데이트됨 (새로운 비교 기준)
3 세 번째 수집 데이터 수집 → 두 번째 스냅샷과 비교 → 변경 감지 → 스냅샷 갱신 ✅ 업데이트됨

시나리오 2: 매일 특정 시간 수집 (예: 매일 오전 9시)

날짜 시간 동작 비교 기준
1일차 09:00 첫 수집 → 스냅샷 생성 없음 (첫 수집)
2일차 09:00 수집 → 1일차 9시 데이터와 비교 → Slack 알림 → 스냅샷 갱신 1일차 09:00 데이터
3일차 09:00 수집 → 2일차 9시 데이터와 비교 → Slack 알림 → 스냅샷 갱신 2일차 09:00 데이터

시나리오 3: 일정 간격 수집 (예: 2시간마다)

시간 동작 비교 기준
09:00 프로그램 실행 → 첫 수집 → 스냅샷 생성 → 타이머 시작 없음
11:00 자동 수집 → 09:00 데이터와 비교 → 알림 → 스냅샷 갱신 09:00 데이터
13:00 자동 수집 → 11:00 데이터와 비교 → 알림 → 스냅샷 갱신 11:00 데이터
15:00 자동 수집 → 13:00 데이터와 비교 → 알림 → 스냅샷 갱신 13:00 데이터
핵심 개념 "이전 상태"는 가장 최근 스케줄 수집 시점의 데이터를 의미합니다. 매번 수집할 때마다 스냅샷이 갱신되어 항상 직전 수집 데이터와 비교합니다.

수집 시간 정보

대시보드 상단에 표시되는 수집 시간 정보:

  • 마지막 수집: 가장 최근에 데이터를 가져온 시간 (현재 데이터 시점)
  • 이전 수집: 스냅샷이 마지막으로 갱신된 시간 (비교 기준 시점)

8. Slack 알림

심의 상태가 변경되면 자동으로 Slack으로 알림을 발송합니다.

Slack Webhook 설정

  1. 설정 화면으로 이동 (우측 상단 설정 아이콘)
  2. Slack 알림 섹션에서 Webhook URL 입력
  3. 연결 테스트 버튼으로 정상 작동 확인
  4. 저장
Webhook URL 발급 Slack 워크스페이스 관리자에게 Incoming Webhook URL을 발급받으세요.

알림 발송 시점

다음 조건을 모두 만족할 때 알림이 발송됩니다:

  1. 데이터 수집 완료
  2. 이전 상태(스냅샷)와 현재 상태를 비교하여 변경 감지
  3. 오늘 동일한 항목에 대해 알림을 보내지 않았거나, 상태/승인번호가 다시 변경됨

중복 알림 방지

동일한 심의 항목에 대해 하루에 여러 번 같은 알림을 보내지 않습니다.

  • 알림 발송 이력은 데이터베이스에 저장됨 (7일 후 자동 삭제)
  • 상태 또는 승인번호가 다시 변경되면 새로운 알림 발송

테스트 메시지

설정 화면에서 연결 테스트 버튼을 클릭하면 다음과 같은 테스트 메시지가 발송됩니다:

실제 알림 형식

상태 변경이 감지되면 다음과 같은 형식으로 알림이 발송됩니다:

항목 설명
변경 건수 상태가 변경된 심의 항목 수
감지 시간 변경을 감지한 시간 (현재 수집 시간)
변경 목록 변경된 각 항목의 심의번호, 병원명, 이전→현재 상태
하단 시간 정보 이전 수집 시간과 현재 수집 시간 비교 안내
주의 스케줄 수집을 설정하지 않으면 스냅샷이 갱신되지 않아 오래된 데이터와 비교하게 됩니다. 적절한 수집 간격을 설정하세요.

9. 설정

우측 상단의 설정 아이콘을 클릭하여 설정 화면으로 이동합니다.

수집 스케줄 설정

매일 특정 시간

  • 매일 지정한 시간에 1회 자동 수집
  • 예: 매일 오전 9시, 매일 오후 6시
  • 시간 형식: HH:mm (24시간 형식)

일정 간격

  • 지정한 시간 간격으로 반복 수집
  • 선택 가능 간격: 30분, 1시간, 2시간, 3시간, 4시간, 5시간, 6시간
  • 프로그램 실행 시 첫 수집 후 간격 타이머 시작
권장 설정 업무 시간 시작 전에 한 번 수집하는 것을 권장합니다 (예: 매일 오전 9시). 변경 사항을 실시간으로 추적하려면 2시간 간격 수집을 사용하세요.

기타 설정

항목 설명
KDA 로그인 정보 비밀번호 변경 및 로그인 상태 확인
테마 라이트/다크 모드 선택

10. 업데이트

프로그램은 자동으로 업데이트를 확인하고 설치합니다.

업데이트 과정

  1. 앱 실행 시 자동으로 업데이트 확인 (GitHub 릴리즈)
  2. 새 버전이 있으면 백그라운드에서 다운로드 (작업 표시줄 진행률 표시)
  3. "업데이트 준비 완료" 토스트 알림 표시
  4. 앱 종료 시 자동으로 설치
  5. 다음 실행 시 새 버전 적용
참고 업데이트는 10분마다 주기적으로 체크됩니다. 업데이트 설치를 위해 앱을 완전히 종료해주세요.

11. FAQ

Q. "이전 상태"가 업데이트되지 않습니다.

A. 이전 상태(스냅샷)는 스케줄 자동 수집이 실행될 때마다 갱신됩니다. 설정에서 스케줄이 올바르게 설정되어 있는지, 프로그램이 실행 중인지 확인하세요.

Q. 처음 수집 시 모든 항목이 "변경됨"으로 표시됩니다.

A. 처음 수집 시에는 이전 데이터가 없어 스냅샷이 자동으로 생성됩니다. 두 번째 수집부터 정상적으로 변경 감지가 작동합니다.

Q. Slack 알림이 오지 않습니다.

A. 다음 사항을 확인하세요:

  • 설정에서 Webhook URL이 올바르게 입력되었는지
  • "연결 테스트" 버튼으로 정상 작동하는지
  • 실제로 상태가 변경된 항목이 있는지 (파란색 배경)
  • 오늘 이미 동일한 알림을 보냈는지 (중복 방지)

Q. 데이터 수집이 실패합니다.

A. 다음을 확인하세요:

  • 인터넷 연결 상태
  • KDA 로그인 정보가 올바른지 (설정에서 재입력)
  • dentalad.or.kr 사이트가 정상 작동하는지

Q. 스케줄이 작동하지 않습니다.

A. 스케줄은 프로그램이 실행 중일 때만 작동합니다. 컴퓨터가 절전 모드이거나 프로그램이 종료되면 수집되지 않습니다.
