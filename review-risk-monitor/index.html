Review Risk Monitor

악성 리뷰 자동 감지 및 알림 시스템

1. 개요

이 프로그램은 네이버/카카오/구글 플랫폼에서 고객사의 리뷰를 자동으로 수집하고, Claude AI를 이용하여 악성 리뷰를 감지하여 Slack으로 즉시 알림을 보내는 Windows 데스크톱 애플리케이션입니다.

주요 기능

  • 3대 플랫폼(네이버/카카오/구글) 리뷰 자동 수집
  • Claude AI 기반 악성 리뷰 자동 분석 및 판정
  • 악성 리뷰 감지 시 Slack 실시간 알림
  • 스케줄 기반 자동 수집 (매일 특정 시간 또는 일정 간격)
  • 거래처별 리뷰 관리 및 필터링
  • 지식 DB를 통한 AI 학습 데이터 관리
  • 자동 업데이트

시스템 요구사항

항목 요구사항
운영체제 Windows 10/11 (64bit)
RAM 4GB 이상 권장
네트워크 인터넷 연결 필수 (리뷰 수집, AI 분석, Slack 알림)
필수 API 키 - Outscraper API 키 (구글 리뷰 수집)
- Claude AI API 키 (악성 분석)
- Slack Webhook URL (알림)

2. 설치

관리자로부터 전달받은 설치 파일을 실행합니다.

  1. GitHub Releases에서 최신 버전의 설치 파일 다운로드
  2. 설치 파일(group-d-review-risk-monitor-setup-x.x.x.exe) 실행
  3. 설치 완료 후 자동으로 프로그램 실행
참고 프로그램은 사용자 폴더에 설치되며, 관리자 권한이 필요하지 않습니다. 앱 실행 시 자동으로 업데이트를 확인합니다.

PC 인증 (첫 실행 시)

인증되지 않은 PC에서 처음 실행하면 Machine Key 인증 화면이 표시됩니다.

인증 방법

  1. 화면에 표시된 32자리 Machine Key를 복사합니다
  2. 관리자에게 키를 전달합니다
  3. 관리자가 GitHub 저장소에 키를 등록하면 앱을 다시 시작합니다
  4. 인증 완료 후 메인 화면으로 진입합니다
주의 Machine Key는 PC의 고유 하드웨어 정보로 생성되므로 PC마다 다릅니다. 새 PC에서 실행하거나 하드웨어를 변경하면 재인증이 필요합니다.

3. 초기 설정

인증 완료 후 처음 사용 시 필수 API 키를 입력해야 합니다.

필수 설정 항목

우측 상단 설정 아이콘 → 앱 설정 탭에서 다음 항목을 입력합니다:

항목 용도 발급 방법
Outscraper API 키 구글 리뷰 수집 outscraper.com에서 회원가입 후 API 키 발급
Claude AI API 키 악성 리뷰 AI 분석 console.anthropic.com에서 API 키 발급
Slack Webhook URL 악성 리뷰 알림 Slack 워크스페이스 관리자에게 Incoming Webhook URL 발급 요청
보안 모든 API 키는 Windows 자격증명 관리자에 암호화되어 저장됩니다. 프로그램 외부에서 접근할 수 없습니다.

Outscraper API 키 발급

  1. outscraper.com 접속 및 회원가입
  2. Dashboard → API → API Keys 메뉴 이동
  3. "Create New API Key" 클릭하여 키 발급
  4. 생성된 키를 복사하여 앱 설정에 입력

Claude AI API 키 발급

  1. console.anthropic.com 접속 및 회원가입
  2. Settings → API Keys 메뉴 이동
  3. "Create Key" 클릭하여 키 발급
  4. 생성된 키를 복사하여 앱 설정에 입력
주의 Claude AI API는 사용량에 따라 과금됩니다. 요금제 및 사용량 한도를 확인하세요.

4. 대시보드

설정 완료 후 메인 화면에서 거래처별 리뷰를 확인할 수 있습니다.

화면 구성

  • 좌측 패널: 거래처 목록 (검색, 필터, 통계)
  • 우측 패널: 3분할 리뷰 뷰어 (네이버/카카오/구글)
  • 상단 헤더: 설정 버튼, 테마 토글, 홈 버튼
  • 하단 상태바: 스케줄 정보 (다음 수집 시간, 마지막 수집 시간)

거래처 목록 패널

기능

  • 검색: 거래처명으로 검색
  • 즐겨찾기 필터: 즐겨찾기한 거래처만 표시
  • 통계 표시:
    • 악성 (빨간색): AI가 악성으로 판정한 리뷰 수
    • 정상 (초록색): 정상으로 판정한 리뷰 수
    • 수집 (파란색): 수집은 되었으나 분석되지 않은 리뷰 수
    • 전체 (회색): 전체 리뷰 수
  • 진행 중 표시: 번개 아이콘으로 현재 수집/분석 중인 거래처 표시

리뷰 뷰어 (3분할)

플랫폼별 리뷰 목록

각 리뷰 카드에는 다음 정보가 표시됩니다:

  • 작성자: 리뷰 작성자 이름
  • 평점: 별점 (1~5점)
  • 작성일: 리뷰 작성 날짜
  • 리뷰 내용: 실제 리뷰 텍스트
  • AI 분석 결과:
    • 악성도 점수 (0~100)
    • 판정 (정상/악성)
    • 악성 유형 (욕설, 허위 정보, 경쟁사 조작 등)
    • 판단 사유
  • 브라우저에서 열기: 원본 리뷰 페이지로 이동
팁 거래처 목록에서 거래처를 클릭하면 우측 리뷰 뷰어에 해당 거래처의 리뷰가 표시됩니다. 각 플랫폼 탭을 클릭하여 네이버/카카오/구글 리뷰를 확인할 수 있습니다.

5. 거래처 관리

설정 → 거래처 관리 탭에서 모니터링할 거래처를 추가/수정/삭제할 수 있습니다.

거래처 추가

  1. [+ 거래처 추가] 버튼 클릭
  2. 거래처명 입력 (필수)
  3. 플랫폼 URL 입력 (선택):
    • 네이버 플레이스 URL (예: https://m.place.naver.com/place/...)
    • 카카오맵 URL (예: https://place.map.kakao.com/...)
    • 구글 맵 URL (예: https://www.google.com/maps/place/...)
  4. 메모 입력 (선택)
  5. 저장 버튼 클릭
주의 URL은 자동으로 유효성을 검증합니다. 올바른 형식의 URL을 입력하세요. 잘못된 URL은 저장되지 않습니다.

거래처 수정/삭제

  • 수정: 테이블에서 행을 클릭하면 편집 대화상자가 열립니다
  • 삭제: 편집 대화상자에서 [삭제] 버튼 클릭
  • 즐겨찾기: 별 아이콘을 클릭하여 즐겨찾기 추가/제거

엑셀 임포트/엑스포트

데이터 가져오기 (Import)

  1. [양식 다운로드] 버튼 클릭하여 엑셀 템플릿 다운로드
  2. 엑셀 파일 작성:
    • 거래처명 (필수)
    • 네이버 플레이스 URL (선택)
    • 카카오맵 URL (선택)
    • 구글 맵 URL (선택)
    • 메모 (선택)
  3. [데이터 가져오기] 버튼 클릭하여 작성한 엑셀 파일 선택
  4. 자동으로 데이터 검증 및 저장

데이터 내보내기 (Export)

  1. [내보내기] 버튼 클릭
  2. 모든 거래처 데이터가 엑셀 파일로 저장됩니다 (ID 포함)
  3. 파일명 형식: [GD-RRM] 거래처목록_YYYYMMDD_HHmm.xlsx
팁 여러 거래처를 한 번에 추가하려면 엑셀 임포트 기능을 사용하세요. 양식 다운로드로 받은 파일에 데이터를 작성한 후 가져오기하면 빠르게 등록할 수 있습니다.

6. 지식 DB 관리

설정 → 지식 DB 탭에서 AI가 참고하는 악성 리뷰 특징 데이터베이스를 관리할 수 있습니다.

지식 DB란?

지식 DB는 Claude AI가 리뷰를 분석할 때 참고하는 악성 리뷰 사례 및 특징을 저장한 데이터베이스입니다. AI는 이 데이터를 학습하여 더 정확하게 악성 리뷰를 판별합니다.

데이터 구조

항목 설명
ID 고유 식별번호 (자동 생성)
악성유형 욕설, 허위 정보, 경쟁사 조작, 협박, 비방, 기타
심각도 1(낮음) ~ 5(매우 높음)
리뷰 내용 실제 악성 리뷰 텍스트 예시
판단 사유 왜 이 리뷰가 악성인지 설명
플랫폼 네이버/카카오/구글 (선택)
거래처 관련 거래처 (선택)

기능

  • 악성유형 필터: 특정 악성 유형만 표시
  • 심각도 정렬: 오름차순/내림차순/기본 순서
  • 행 확장: 테이블 행을 클릭하면 상세 정보(판단 사유, 거래처, 플랫폼 등) 표시
  • 엑셀 임포트: 양식 다운로드 → 데이터 작성 → 가져오기
팁 지식 DB에 다양한 악성 리뷰 사례를 추가할수록 AI의 판별 정확도가 높아집니다. 실제로 발생한 악성 리뷰를 지식 DB에 등록하여 학습 데이터로 활용하세요.

7. 앱 설정

설정 → 앱 설정 탭에서 스크래핑, AI 분석, 알림, 수집 스케줄을 설정할 수 있습니다.

1. 스크래핑 설정

플랫폼별로 수집할 리뷰 개수를 설정합니다.

플랫폼 설정 기본값
네이버 페이지 수 (1~5) 1페이지 (약 20개 리뷰)
카카오 페이지 수 (1~5) 1페이지 (약 20개 리뷰)
구글 수집 개수 (5/10/15/20/30/50/100) 5개
주의 구글 리뷰는 Outscraper API를 사용하므로 사용량에 따라 과금됩니다. 수집 개수가 많을수록 비용이 증가합니다.

2. Outscraper API 설정

  • API 키: Outscraper API 키 입력 (구글 리뷰 수집용)
  • 입력 후 자동으로 암호화 저장

3. Claude AI 설정

  • API 키: Claude AI API 키 입력 (악성 리뷰 분석용)
  • 모델 선택:
    • Haiku 4.5 (기본): 가장 빠르고 저렴, 일반적인 분석에 적합
    • Sonnet 4.5: 중간 성능, 복잡한 분석에 적합
    • Opus 4.5: 최고 성능, 가장 비쌈
권장 설정 일반적인 사용에는 Haiku 4.5 모델을 권장합니다. 비용 대비 성능이 우수하며, 악성 리뷰 판별에 충분한 정확도를 제공합니다.

4. Slack 알림 설정

  • Webhook URL: Slack Incoming Webhook URL 입력
  • 연결 테스트: 테스트 메시지 발송하여 정상 작동 확인

5. 데이터 수집 스케줄

자동 수집 모드를 설정합니다.

매일 특정 시간 (Daily)

  • 매일 지정한 시간에 1회 자동 수집
  • 시간 형식: HH:mm (24시간 형식)
  • 예: 매일 오전 9시 → 09:00

일정 간격 (Interval)

  • 지정한 시간 간격으로 반복 수집
  • 반복 주기(분) 입력 (예: 120분 = 2시간마다)
  • 프로그램 실행 시 첫 수집 후 타이머 시작
자동화 스케줄을 설정하면 프로그램이 실행 중일 때 자동으로 리뷰를 수집하고 분석합니다. 악성 리뷰가 감지되면 즉시 Slack으로 알림이 발송됩니다.

8. 관리자 모드

관리자 모드는 일반 사용자에게는 숨겨진 고급 기능을 제공합니다.

활성화 방법

설정 화면에서 ESC 키를 5초 내에 10회 연타하면 관리자 모드가 활성화됩니다.

주의 관리자 모드는 전문 사용자를 위한 기능입니다. 잘못된 사용은 데이터 손실이나 예상치 못한 동작을 유발할 수 있습니다.

활성화 시 추가되는 기능

1. 거래처 목록 [실행] 버튼

  • 대시보드의 거래처 목록 상단에 [실행] 버튼이 표시됩니다
  • 현재 필터링된 거래처(검색어, 즐겨찾기 적용)를 대상으로 즉시 수집/분석/알림을 실행합니다
  • 실행 전 확인 다이얼로그에서 대상 거래처 목록을 확인할 수 있습니다

2. 리뷰 패널 새로고침 버튼

  • 각 리뷰 패널 헤더에 새로고침 버튼이 표시됩니다
  • 해당 플랫폼의 리뷰를 즉시 재수집합니다

3. 프롬프트 편집 탭

  • 설정 화면에 프롬프트 탭이 추가됩니다
  • AI 분석에 사용되는 시스템 프롬프트를 직접 수정할 수 있습니다
  • 기본값으로 복원 버튼으로 원래 프롬프트로 되돌릴 수 있습니다
경고 프롬프트를 잘못 수정하면 AI 분석 결과가 부정확해질 수 있습니다. 변경 전 반드시 백업하고, 테스트 후 적용하세요.

비활성화 방법

관리자 모드를 비활성화하려면 ESC 키를 다시 10회 연타하세요.

9. FAQ

Q. 거래처를 추가했는데 리뷰가 안 보여요.

A. 거래처 추가 직후에는 리뷰 수집이 이루어지지 않습니다. 다음 중 하나를 수행하세요:

  • 스케줄 수집이 실행될 때까지 대기
  • 관리자 모드를 활성화하고 [실행] 버튼으로 즉시 수집

Q. 악성 리뷰는 어떻게 판정하나요?

A. Claude AI가 다음 기준으로 리뷰를 분석합니다:

  • 욕설 및 비속어 사용
  • 허위 정보 포함 여부
  • 경쟁사 조작 의심
  • 협박성 문구
  • 비방 및 명예훼손
  • 지식 DB에 등록된 악성 패턴과의 유사도

Q. Slack 알림이 오지 않아요.

A. 다음 사항을 확인하세요:

  • 설정 → 앱 설정 → Slack 알림에서 Webhook URL이 올바르게 입력되었는지
  • "연결 테스트" 버튼으로 정상 작동하는지 확인
  • 실제로 악성 리뷰가 감지되었는지 확인 (대시보드에서 빨간색 배지 확인)
  • 스케줄 수집이 설정되어 있고 실행되었는지 확인

Q. 자동 수집은 언제 실행되나요?

A. 설정 → 앱 설정 → 데이터 수집에서 설정한 스케줄에 따라 자동 실행됩니다. 하단 상태바에서 다음 수집까지 남은 시간을 확인할 수 있습니다.

Q. 여러 거래처를 한 번에 추가할 수 있나요?

A. 네, 설정 → 거래처 관리 → 양식 다운로드로 엑셀 양식을 받아 작성 후 데이터 가져오기를 하면 됩니다.

Q. 지식 DB는 어떻게 활용하나요?

A. 실제로 발생한 악성 리뷰 사례를 지식 DB에 등록하면 AI가 학습하여 더 정확하게 악성 리뷰를 판별합니다. 악성 유형, 심각도, 판단 사유를 함께 입력하면 효과적입니다.

Q. 관리자 모드는 언제 사용하나요?

A. 다음과 같은 상황에서 사용합니다:

  • 거래처 추가 직후 즉시 리뷰를 수집하고 싶을 때
  • 특정 플랫폼의 리뷰만 재수집하고 싶을 때
  • AI 프롬프트를 커스터마이징하고 싶을 때

Q. 데이터는 어디에 저장되나요?

A. 모든 데이터는 사용자 폴더에 저장됩니다:

C:\Users\[사용자명]\AppData\Roaming\group-d-review-risk-monitor
  • database.sqlite: 거래처, 리뷰, 지식 DB 데이터
  • settings.json: 앱 설정 (스케줄, 테마 등)
  • API 키: Windows 자격증명 관리자 (별도 암호화 저장)
